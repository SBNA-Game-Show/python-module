Welcome to Samskrita Bharati's Dictionary Module

Application Features

The Sanskrit Dictionary Module is designed to support Sanskrit learners, educators, and developers in exploring and analyzing Sanskrit texts. The application can ingest content from various sources, such as uploaded documents and web-based content. It extracts, processes, and tokenizes both English and Sanskrit text, generating linguistic metadata that is subsequently stored in MongoDB for efficient retrieval, analysis, and educational use.

This application stores its data in the MongoDB database WordComplex, within a dedicated collection also named tokenized-stories. It supports flexible access to the stored data through API requests. Please refer to the API documentation for details on the available endpoints for developers.

Libraries Used :

  • Web-Based Content Extraction:
    • Flask Requests

      Used to establish HTTP connections and retrieve content from external web resources. The library enables the application to send requests, receive responses, and collect raw data for further processing.

    • Beautiful Soup

      Used for parsing and navigating HTML documents. It extracts relevant textual content, removes unnecessary markup, and structures the data for downstream linguistic analysis and tokenization.

  • Tokenizing English Passage:
    • Spacy

      This application leverages spaCy's en_core_web_sm language model to perform linguistic analysis on English passages. The complete passage is processed by the model, which identifies the part of speech (POS) for each token. The resulting tokens, along with their corresponding lemmas and POS tags, are then stored in a structured array for subsequent analysis, dictionary lookups, synonym and antonym retrieval, and other language processing tasks.

  • Adding Synonym and Antonyms to Tokenized English Words:
    • NLTK WordNet

      This application leverages NLTK's WordNet lexical database to enrich tokenized English words with their corresponding synonyms and antonyms. Using the part-of-speech information generated during tokenization, the system performs targeted WordNet lookups to retrieve semantically relevant word relationships. The extracted lexical data is then stored alongside each token, enabling enhanced dictionary functionality, vocabulary exploration, and language-learning support.

  • Tokenizing Sanskrit Passage:
    • Dharmamitra

      The Sanskrit tokenization process follows these steps:

      • A POST request is sent to Dharmamitra's API.
      • The response output is extracted and normalized to match the English tokenization process.
  • Reading PDF Documents:
    • English PDF Extraction
      • PyMuPDF

        PyMuPDF is used to extract embedded text directly from PDF documents. The library parses document pages, retrieves textual content while preserving reading order, and prepares the extracted data for downstream tokenization, linguistic analysis, and database storage.

    • Sanskrit PDF Extraction
      • PyMuPDF + Tesseract OCR (sanskrit)

        As Sanskrit texts often contain complex Devanagari characters, ligatures, and font encodings that cannot always be extracted accurately using standard text-based PDF parsing techniques. To address this challenge, the application combines **PyMuPDF** and **Tesseract OCR (Sanskrit/Devanagari language model)** to perform image-based text extraction. PyMuPDF is used to identify and extract page regions or render PDF content as high-resolution images, while Tesseract OCR processes these images to recognize Devanagari characters and convert them into Unicode Sanskrit text. The extracted content is subsequently cleaned, normalized, and structured to ensure consistency before undergoing tokenization, lexical enrichment, and further linguistic analysis.

  • Reading JSON:
    • Python Built-in JSON/File Handling

      To parse the JSON document, the application uses Python’s built-in file system reader along with the JSON module to load and process the file. The extracted data is then prepared for downstream linguistic analysis and tokenization.

  • Reading Microsoft Word:
    • Python-docx

      To parse Microsoft Word documents, the application uses Python’s `python-docx` library to load and process the file contents. The extracted text is then prepared for downstream tokenization and further language processing tasks.

Data Flow :

  • Web-Based:

    • This application follows a structured pipeline to complete the tokenization process, including extracting data from web sources, processing and tokenizing the extracted content, and persisting the resulting linguistic data into MongoDB.

  • Read PDF:

    • This application follows a structured pipeline for extracting data from PDF files, normalizing and tokenizing the extracted content, and persisting the resulting linguistic data into MongoDB.

  • Read JSON:

    • This application processes JSON files by validating the provided key-value structure, extracting passage data, normalizing the content, tokenizing the text, and storing the resulting linguistic information in MongoDB.

  • Read Microsoft Word Document Work Flow:

    • Microsoft Word documents are processed using Python's python-docx library to extract textual content. The extracted data is normalized, tokenized, analyzed linguistically, and stored in MongoDB for downstream applications.
