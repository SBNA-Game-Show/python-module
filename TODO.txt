1. System Enhancements Required
- Implement a Request Queue Management System
- Due to limited memory availability, only one POST request should be processed at a time.
- Implement a queue-based processing mechanism using a First-In-First-Out (FIFO) approach.
- Incoming POST requests should be added to the queue and executed sequentially.
- The next request should begin processing only after the previous request completes, regardless of whether the response is successful or failed.
- The queue system should handle:
- Request tracking
- Processing status
- Failure handling
- Automatic execution of the next queued request
2. Implement CI/CD Pipeline
- Add an automated CI/CD pipeline to improve deployment reliability.
- The pipeline should include:
- Automated test execution
- Code quality checks
- Build process
- Docker image creation
- Deployment automation
- Environment-specific configuration handling
3. Implement Token-Based Authentication
- Secure application endpoints by adding token-based authentication.
- Protect all sensitive API endpoints from unauthorized access.
